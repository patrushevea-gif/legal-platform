# ADR 0004: Web and API demo integration

## Status

Accepted

## Context

К этому шагу в проекте уже были:

- auth и persistence на API
- очередь юристов и backend workflow
- отдельный web-shell без реальной связи с backend

Это означало, что продукт всё ещё нельзя показать как сквозной demo.

## Decision

- добавить CORS для локального запуска `web -> api`
- открыть endpoint со списком legal team для назначения исполнителей
- связать web с реальными API-эндпоинтами логина, создания заявок и очереди
- использовать demo-сессии из seed-пользователей как baseline для product demo

## Consequences

Появляется первый end-to-end сценарий на уровне интерфейса:

- пользователь входит в систему
- бизнес-заказчик создаёт заявку
- юрист открывает очередь
- назначает исполнителя и меняет статус

Следующий шаг:

- вынести API-клиент и UI-блоки в отдельные модули
- добавить форму логина без seed shortcuts
- подключить server-side data loading и защищённые маршруты
