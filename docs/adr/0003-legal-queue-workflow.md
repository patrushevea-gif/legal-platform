# ADR 0003: Legal queue workflow baseline

## Status

Accepted

## Context

После появления auth и persistence API всё ещё не покрывал ключевую операционную задачу:

- юрист не мог взять заявку в работу
- нельзя было назначить исполнителя
- не было API-представления очереди для legal team

Без этого web-интерфейс юриста оставался чисто декоративным.

## Decision

- расширить `legal_requests` полями назначения исполнителя
- добавить queue endpoint для юридической команды
- добавить отдельные действия `assign` и `status`
- считать назначение началом работы и автоматически переводить `new -> in_review`
- разрешить юристу менять статусы только у своих или ещё не назначенных заявок

## Consequences

Система получает первый рабочий legal-операционный контур:

- бизнес создаёт заявку
- юрист видит её в очереди
- назначает исполнителя
- переводит в нужный статус

Следующий логичный шаг:

- web-форма подачи заявки
- связка web с auth/API
- фильтры, SLA и приоритетные представления очереди
